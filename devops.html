This Week's Tasks

Step 01

  1. Install Docker 
    sudo apt-get update
sudo apt-get install docker.io
  2. Create Container 
    docker run -it --name my-container ubuntu
  3. Access your container 
    docker exec -it my-container bash
  4. Clone your code into the container 
    git clone https://github.com/yourusername/your-repo.git
  5. Create image from container 
    docker commit my-container my-image
  6. Push your Docker image to Docker Hub 
    docker login
docker tag my-image yourusername/my-image
docker push yourusername/my-image
  7. Delete container and local images 
    docker rm my-container
docker rmi my-image
  8. Pull your images 
    docker pull yourusername/my-image
  9. Create a container and start it on port 80:80 
    docker run -d -p 80:80 yourusername/my-image
  10. Access your website 
    http://localhost

Step 02

  1. Make changes in your GitHub code
  2. Start new container 
    docker run -it --name new-container ubuntu
  3. Clone your code in this new container and create image and push 
    git clone https://github.com/yourusername/your-repo.git
docker commit new-container new-image
docker tag new-image yourusername/new-image
docker push yourusername/new-image

Jenkins Pipeline

  1. Delete existing image and container 
    docker rm existing-container
docker rmi existing-image

FAQs

DevOps is a cultural philosophy, practice, and set of tools that increase an organization's ability to deliver applications and services at high velocity.

CI/CD automates the software delivery process, allowing for faster and more reliable releases.

Infrastructure as Code (IaC) is a key DevOps practice that involves managing infrastructure using code and automation, rather than manual processes.
